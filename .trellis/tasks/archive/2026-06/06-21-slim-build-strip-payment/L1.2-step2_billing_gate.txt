## 任务步骤

### 1. 抽 BillingGate 接口
在 `backend/internal/handler/billing_gate.go`（新文件，无 build tag）定义：
```go
package handler

import (
	"context"
	"github.com/Wei-Shaw/sub2api/internal/service"
)

// BillingGate 是 billing 拦截的抽象，供 full/slim 分版实现
type BillingGate interface {
	CheckEligibility(ctx context.Context, subscription *service.UserSubscription, apiKey *service.APIKey, user *service.User) (bool, string, error)
}
```

### 2. 实现 full 版（调现有 BillingCacheService）
`backend/internal/handler/billing_gate_full.go`（`//go:build !slim`）：
```go
//go:build !slim

package handler

import (
	"context"
	"github.com/Wei-Shaw/sub2api/internal/repository"
	"github.com/Wei-Shaw/sub2api/internal/service"
)

type billingGateFull struct {
	billingCache *repository.BillingCacheService
}

func NewBillingGate(billingCache *repository.BillingCacheService) BillingGate {
	return &billingGateFull{billingCache: billingCache}
}

func (g *billingGateFull) CheckEligibility(ctx context.Context, subscription *service.UserSubscription, apiKey *service.APIKey, user *service.User) (bool, string, error) {
	return g.billingCache.CheckBillingEligibility(ctx, subscription, apiKey, user)
}
```

### 3. 实现 slim 版（no-op 恒放行）
`backend/internal/handler/billing_gate_slim.go`（`//go:build slim`）：
```go
//go:build slim

package handler

import (
	"context"
	"github.com/Wei-Shaw/sub2api/internal/repository"
	"github.com/Wei-Shaw/sub2api/internal/service"
)

type billingGateSlim struct{}

func NewBillingGate(_ *repository.BillingCacheService) BillingGate {
	return &billingGateSlim{}
}

func (g *billingGateSlim) CheckEligibility(_ context.Context, _ *service.UserSubscription, _ *service.APIKey, _ *service.User) (bool, string, error) {
	// slim: 纯 API key 鉴权，不做余额/配额拦截，恒放行
	return true, "", nil
}
```

### 4. 改造 12 处调用点
**逐一找到每个调用点**，把：
```go
eligible, reason, err := h.billingCacheService.CheckBillingEligibility(ctx, subscription, apiKey, user)
```
改为：
```go
eligible, reason, err := h.billingGate.CheckEligibility(ctx, subscription, apiKey, user)
```

注意事项：
- 变量名可能是 `h.billingCache`、`billingCacheService` 等，统一改为 `h.billingGate`。
- handler 结构体需加 `billingGate BillingGate` 字段（12 个 handler 文件，可能有些共享一个结构体）。
- handler 构造函数需接收 `billingGate BillingGate` 参数（可能从 `NewBillingGate(billingCacheService)` 注入）。
- **保留调用点上下文不变**（调用前后的并发槽、订阅读取、错误处理逻辑完全不动），只替换调用对象。

### 5. 验证

两版均编译通过，且 `billing_gate_full.go` 与 `billing_gate_slim.go` 互斥（不会同时编译）。
